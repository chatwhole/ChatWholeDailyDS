## Key Concepts Summary

### SQL Statement Categories

| Category | Full Name | Commands | Purpose |
|----------|-----------|----------|---------|
| DDL | Data Definition Language | CREATE, ALTER, DROP, TRUNCATE | Define database structure |
| DML | Data Manipulation Language | SELECT, INSERT, UPDATE, DELETE | Manipulate data |
| DCL | Data Control Language | GRANT, REVOKE | Control access permissions |
| TCL | Transaction Control Language | COMMIT, ROLLBACK, SAVEPOINT | Manage transactions |

### Join Types Comparison

| Join Type | Returns | Use Case |
|-----------|---------|----------|
| INNER JOIN | Only matching rows | Finding related data |
| LEFT JOIN | All left + matching right | Preserving left table data |
| RIGHT JOIN | All right + matching left | Preserving right table data |
| FULL JOIN | All rows from both | Finding unmatched in either |
| CROSS JOIN | Cartesian product | Generating combinations |
| SELF JOIN | Table joined to itself | Comparing rows within same table |

### Aggregate Functions

| Function | Purpose | Example |
|----------|---------|---------|
| COUNT() | Count rows | `COUNT(*)`, `COUNT(DISTINCT col)` |
| SUM() | Sum numeric values | `SUM(amount)` |
| AVG() | Average numeric values | `AVG(price)` |
| MIN() | Find minimum value | `MIN(order_date)` |
| MAX() | Find maximum value | `MAX(revenue)` |

### Window Functions

| Function | Purpose | Example |
|----------|---------|---------|
| ROW_NUMBER() | Sequential numbering | `ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary DESC)` |
| RANK() | Rank with gaps | `RANK() OVER (ORDER BY score DESC)` |
| DENSE_RANK() | Rank without gaps | `DENSE_RANK() OVER (ORDER BY score DESC)` |
| LEAD() | Access next row | `LEAD(amount, 1) OVER (ORDER BY date)` |
| LAG() | Access previous row | `LAG(amount, 1) OVER (ORDER BY date)` |
| NTILE() | Divide into buckets | `NTILE(4) OVER (ORDER BY salary)` |
| FIRST_VALUE() | First value in window | `FIRST_VALUE(name) OVER (ORDER BY date)` |
| LAST_VALUE() | Last value in window | `LAST_VALUE(name) OVER (ORDER BY date ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING)` |

### NULL Handling Functions

| Function | Purpose | Example |
|----------|---------|---------|
| COALESCE() | Return first non-NULL | `COALESCE(nickname, name, 'Unknown')` |
| NULLIF() | Return NULL if equal | `NULLIF(a, b)` — returns NULL if a = b |
| IS NULL | Check for NULL | `WHERE email IS NULL` |
| IS NOT NULL | Check for not NULL | `WHERE email IS NOT NULL` |
| NVL() | Replace NULL (Oracle) | `NVL(commission, 0)` |
| IFNULL() | Replace NULL (MySQL) | `IFNULL(commission, 0)` |
| ISNULL() | Replace NULL (SQL Server) | `ISNULL(commission, 0)` |

## Code Examples

### CREATE TABLE with Constraints

```sql
CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    first_name TEXT NOT NULL,
    last_name TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    salary REAL CHECK (salary > 0),
    department TEXT DEFAULT 'General',
    hire_date DATE DEFAULT CURRENT_DATE,
    is_active BOOLEAN DEFAULT TRUE
);
```

### INSERT with Multiple Rows

```sql
INSERT INTO employees (id, first_name, last_name, email, salary, department)
VALUES
    (1, 'Alice', 'Johnson', 'alice@company.com', 75000, 'Engineering'),
    (2, 'Bob', 'Smith', 'bob@company.com', 62000, 'Marketing'),
    (3, 'Carol', 'Williams', 'carol@company.com', 85000, 'Engineering');
```

### SELECT with WHERE and ORDER BY

```sql
SELECT first_name, last_name, salary, department
FROM employees
WHERE department = 'Engineering'
  AND salary > 70000
ORDER BY salary DESC;
```

### JOIN Example

```sql
SELECT 
    o.id AS order_id,
    c.name AS customer_name,
    o.order_date,
    o.total
FROM orders o
INNER JOIN customers c ON o.customer_id = c.id
WHERE o.order_date >= '2024-01-01'
ORDER BY o.order_date DESC;
```

### GROUP BY with HAVING

```sql
SELECT 
    department,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 60000
ORDER BY avg_salary DESC;
```

### Window Function for Ranking

```sql
SELECT 
    first_name,
    last_name,
    department,
    salary,
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank,
    RANK() OVER (ORDER BY salary DESC) AS overall_rank
FROM employees;
```

### Subquery Example

```sql
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees)
ORDER BY salary DESC;
```

### CTE for Complex Queries

```sql
WITH department_stats AS (
    SELECT 
        department,
        COUNT(*) AS emp_count,
        AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department
)
SELECT 
    e.first_name,
    e.last_name,
    e.salary,
    d.avg_salary AS dept_avg_salary,
    e.salary - d.avg_salary AS salary_diff
FROM employees e
INNER JOIN department_stats d ON e.department = d.department;
```

### CASE Expression

```sql
SELECT 
    first_name,
    last_name,
    salary,
    CASE 
        WHEN salary >= 80000 THEN 'Senior'
        WHEN salary >= 60000 THEN 'Mid-Level'
        ELSE 'Junior'
    END AS salary_band
FROM employees
ORDER BY salary DESC;
```

### COALESCE for NULL Handling

```sql
SELECT 
    name,
    COALESCE(nickname, name, 'Unknown') AS display_name,
    COALESCE(phone, 'No phone') AS contact_phone
FROM customers;
```

### Creating an Index

```sql
CREATE INDEX idx_employees_department ON employees(department);
CREATE INDEX idx_orders_customer_date ON orders(customer_id, order_date);
CREATE UNIQUE INDEX idx_customers_email ON customers(email);
```

### Recursive CTE for Hierarchical Data

```sql
WITH RECURSIVE org_chart AS (
    -- Base case: top-level managers
    SELECT id, name, manager_id, 1 AS level
    FROM employees
    WHERE manager_id IS NULL
    
    UNION ALL
    
    -- Recursive case: direct reports
    SELECT e.id, e.name, e.manager_id, oc.level + 1
    FROM employees e
    INNER JOIN org_chart oc ON e.manager_id = oc.id
)
SELECT * FROM org_chart ORDER BY level, name;
```

## Frequently Asked Questions

### How long does it take to complete the SQL course?
The full 70-lesson SQL course can be completed in 8-12 weeks at a pace of 1-2 lessons per day.

### Is SQL hard to learn?
No. SQL is one of the most beginner-friendly programming languages. With structured lessons and practice, you can write complex queries within weeks.

### Which SQL dialect should I learn?
Start with [PostgreSQL](https://chatwhole.com/learn/sql/64-postgresql-vs-mysql/) — it has the richest feature set and is widely used in data engineering and analytics.

### What is the difference between WHERE and HAVING?
WHERE filters rows before grouping; HAVING filters grouped results after GROUP BY.

### Can I learn SQL for free?
Yes! All 70 SQL lessons are available for free at [chatwhole.com/learn/sql/](https://chatwhole.com/learn/sql/).

### What is the difference between DELETE and TRUNCATE?
DELETE removes specific rows and can be rolled back; TRUNCATE removes all rows, is faster, and typically cannot be rolled back.
