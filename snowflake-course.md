## Key Concepts Summary

### Snowflake Three-Layer Architecture

| Layer | Description | Scaling |
|-------|-------------|---------|
| Cloud Services | Metadata, security, query optimization | Automatic, serverless |
| Query Processing | Virtual warehouses for SQL execution | Manual or auto-scale |
| Storage | Micro-partitions in cloud blob storage | Automatic expansion |

### Virtual Warehouse Sizes

| Size | vCPU | Memory | Credits/Hour | Max Concurrent Queries |
|------|------|--------|--------------|----------------------|
| X-Small | 1 | 2 GB | 1 | 1 |
| Small | 2 | 4 GB | 2 | 2 |
| Medium | 4 | 8 GB | 4 | 4 |
| Large | 8 | 16 GB | 8 | 8 |
| X-Large | 16 | 32 GB | 16 | 16 |
| 2X-Large | 32 | 64 GB | 32 | 32 |
| 3X-Large | 64 | 128 GB | 64 | 64 |
| 4X-Large | 128 | 256 GB | 128 | 128 |

### Key Snowflake Features

| Feature | Description | Use Case |
|---------|-------------|----------|
| Time Travel | Query data at any point in time | Recovery, audit, debugging |
| Zero-Copy Clone | Instant metadata-only copy | Dev/test, data branching |
| Snowpipe | Automated file ingestion | Continuous data loading |
| Streams | Change data capture | Incremental processing |
| Tasks | Scheduled SQL execution | ETL orchestration |
| Dynamic Data Masking | Column-level security | Sensitive data protection |
| Materialized Views | Pre-computed query results | Dashboard performance |

### Snowflake vs Traditional Data Warehouses

| Aspect | Traditional DW | Snowflake |
|--------|---------------|-----------|
| Scaling | Vertical (bigger machines) | Horizontal (add clusters) |
| Storage/Compute | Tightly coupled | Fully separated |
| Concurrency | Limited | Near-unlimited |
| Maintenance | Manual tuning | Automatic optimization |
| Pricing | Node-based | Per-second compute + storage |

## Code Examples

### Snowflake Architecture: Multi-Cluster Warehouse

```sql
-- Create a multi-cluster warehouse with auto-scaling
CREATE WAREHOUSE analytics_wh
    WAREHOUSE_SIZE = 'xlarge'
    MIN_CLUSTER_COUNT = 1
    MAX_CLUSTER_COUNT = 5
    SCALING_POLICY = 'ECONOMY'
    AUTO_SUSPEND = 300
    AUTO_RESUME = TRUE
    COMMENT = 'Multi-cluster warehouse for analytics';

-- Monitor warehouse utilization
SELECT
    warehouse_name,
    warehouse_size,
    SUM(credits_used) AS total_credits,
    AVG(queued_overload_queries) AS avg_queue_depth
FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
WHERE start_date >= DATEADD(day, -7, CURRENT_DATE())
GROUP BY warehouse_name, warehouse_size
ORDER BY total_credits DESC;
```

### Time Travel & Data Cloning

```sql
-- Query data as it was 2 hours ago
SELECT * FROM sales_data AT (OFFSET => -7200);

-- Query data at a specific timestamp
SELECT * FROM sales_data AT (TIMESTAMP => '2025-01-15 10:30:00'::TIMESTAMP_TZ);

-- Zero-copy clone for dev/test
CREATE TABLE sales_staging CLONE sales_data;

-- Clone with time travel
CREATE TABLE sales_jan2025 CLONE sales_data
    AT (TIMESTAMP => '2025-01-31 23:59:59'::TIMESTAMP_TZ);

-- Restore dropped table
UNDROP TABLE sales_data;
```

### Snowpipe for Automated Ingestion

```sql
-- Create file format
CREATE OR REPLACE FILE FORMAT csv_format
    TYPE = 'CSV'
    FIELD_DELIMITER = ','
    SKIP_HEADER = 1
    NULL_IF = ('NULL', 'null', '')
    EMPTY_FIELD_AS_NULL = TRUE;

-- Create stage
CREATE OR REPLACE STAGE my_s3_stage
    URL = 's3://my-bucket/data/'
    STORAGE_INTEGRATION = my_s3_integration;

-- Create pipe for auto-ingestion
CREATE OR REPLACE PIPE orders_pipe
    AUTO_INGEST = TRUE
    COMMENT = 'Auto-ingest orders from S3'
AS
    COPY INTO orders
    FROM @my_s3_stage/orders/
    FILE_FORMAT = csv_format
    ON_ERROR = 'CONTINUE';

-- Check pipe status
SELECT SYSTEM$PIPE_STATUS('orders_pipe');
```

### Semi-Structured Data Processing

```sql
-- Create table with VARIANT column for JSON
CREATE TABLE json_events (
    event_id VARCHAR,
    event_time TIMESTAMP,
    payload VARIANT
);

-- Query JSON data
SELECT
    event_id,
    payload:event_type::STRING AS event_type,
    payload:user_id::INTEGER AS user_id,
    payload:metadata:browser::STRING AS browser
FROM json_events
WHERE payload:event_type = 'purchase';

-- Flatten nested JSON arrays
SELECT
    event_id,
    f.value AS item
FROM json_events,
LATERAL FLATTEN(input => payload:items) f;

-- Parse and aggregate JSON data
SELECT
    payload:product_category::STRING AS category,
    COUNT(*) AS event_count,
    AVG(payload:amount::FLOAT) AS avg_amount
FROM json_events
GROUP BY category;
```

### Snowpark Python: DataFrames and UDFs

```python
from snowflake.snowpark import Session
from snowflake.snowpark.functions import col, sum as sum_, avg

# Create session
session = Session.builder.configs({
    "account": "your_account",
    "user": "your_user",
    "password": "your_password",
    "warehouse": "ANALYTICS_WH",
    "database": "ANALYTICS_DB"
}).create()

# Read and transform data
df = session.table("sales")
result = df.filter(col("amount") > 1000) \
           .group_by("region") \
           .agg(sum_(col("amount")).alias("total")) \
           .collect()

# Scalar UDF
from snowflake.snowpark.types import FloatType
from snowflake.snowpark.udf import udf

@udf(name="calculate_tax", return_type=FloatType(), input_types=[FloatType()])
def calculate_tax(amount: float) -> float:
    return amount * 0.08

session.udf.register(calculate_tax)
```

### Streams & Tasks for CDC

```sql
-- Create stream on source table
CREATE OR REPLACE STREAM orders_stream
    ON TABLE orders
    SHOW_INITIAL_ROWS = TRUE;

-- Create task to process changes
CREATE OR REPLACE TASK process_order_changes
    WAREHOUSE = 'ETL_WH'
    SCHEDULE = 'USING CRON 0 * * * * UTC'
AS
    MERGE INTO orders_archive t
    USING orders_stream s
    ON t.order_id = s.order_id
    WHEN MATCHED AND s.METADATA$ACTION = 'DELETE' THEN
        DELETE
    WHEN MATCHED AND s.METADATA$ACTION = 'INSERT' THEN
        UPDATE SET t.amount = s.amount, t.status = s.status
    WHEN NOT MATCHED THEN
        INSERT (order_id, amount, status, order_date)
        VALUES (s.order_id, s.amount, s.status, s.order_date);

-- Resume task
ALTER TASK process_order_changes RESUME;
```
