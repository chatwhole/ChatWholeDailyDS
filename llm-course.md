### LLM Model Comparison

| Claude 3.5 | ~200B (est.) | 200K | API only | Long documents |

| Qwen 2 | 0.5B-72B | 128K | Open source | Multilingual |
| Phi-3 | 3.8B-14B | 128K | Open source | Small device |

### Fine-Tuning Methods Comparison

| Method | Trainable Params | Memory | When to Use |
|--------|-----------------|--------|-------------|
| Full Fine-Tuning | 100% | Very High | Large dataset, full control |
| LoRA | 0.1-1% | Medium | Most use cases |
| QLoRA | 0.1-1% | Low | Consumer GPUs |
| Prompt Tuning | <0.01% | Minimal | Quick experiments |
| IA3 | <0.01% | Minimal | Fast adaptation |

### RAG Components

```
Query → Embedding → Vector Search → Retrieve Docs → Context → LLM → Response

Key Components:
- Embedding Model: OpenAI, Cohere, BGE, E5
- Vector DB: Pinecone, Chroma, Weaviate, Milvus, Qdrant
- LLM: GPT-4, Claude, LLaMA, Mistral
- Chunking: Fixed, Semantic, Recursive
```

### Prompt Engineering Techniques

| Technique | Description | Example |
|-----------|-------------|---------|
| Zero-shot | No examples | "Classify this text as positive or negative" |
| Few-shot | Include examples | "Text: Great! → Positive. Text: Bad → Negative. Text: OK → ?" |
| Chain-of-Thought | Step-by-step reasoning | "Let's think step by step..." |
| Self-Consistency | Multiple samples, majority vote | Generate 5 responses, pick most common |
| Tree-of-Thought | Branching reasoning | Explore multiple reasoning paths |

### Basic Text Generation with OpenAI

```python
from openai import OpenAI

client = OpenAI(api_key="your-api-key")

response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Explain transformers in simple terms."}
    ],
    temperature=0.7,
    max_tokens=500
)

print(response.choices[0].message.content)
```

### Fine-Tuning with LoRA

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "meta-llama/Llama-2-7b-hf"
model = AutoModelForCausalLM.from_pretrained(model_name, load_in_4bit=True)
tokenizer = AutoTokenizer.from_pretrained(model_name)

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# Output: trainable params: 4,194,304 || all params: 6,742,609,920 || 0.06%
```

### RAG with Chroma and LangChain

```python
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings
from langchain.chat_models import ChatOpenAI
from langchain.chains import RetrievalQA
from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# Load and split documents
loader = TextLoader("docs.txt")
documents = loader.load()
text_splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
docs = text_splitter.split_documents(documents)

# Create vector store
vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())

# Create QA chain
qa_chain = RetrievalQA.from_chain_type(
    llm=ChatOpenAI(model="gpt-4", temperature=0),
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3})
)

# Query
result = qa_chain.run("What are the main topics covered?")
print(result)
```

### Prompt Template System

```python
def build_prompt(system_prompt, user_input, examples=None, context=None):
    messages = [{"role": "system", "content": system_prompt}]
    
    if context:
        messages.append({"role": "system", "content": f"Context: {context}"})
    
    if examples:
        for ex in examples:
            messages.append({"role": "user", "content": ex["input"]})
            messages.append({"role": "assistant", "content": ex["output"]})
    
    messages.append({"role": "user", "content": user_input})
    return messages

# Usage
prompt = build_prompt(
    system_prompt="You are a data analyst. Analyze the data and provide insights.",
    user_input="What trends do you see in this sales data?",
    examples=[
        {"input": "Sales dropped in Q3", "output": "The Q3 decline suggests seasonal factors..."}
    ],
    context="Monthly sales data for 2024"
)
```

### LLM Evaluation Pipeline

```python
from deepeval import evaluate
from deepeval.metrics import AnswerRelevancyMetric, FaithfulnessMetric
from deepeval.test_case import LLMTestCase

# Define test case
test_case = LLMTestCase(
    input="What is machine learning?",
    actual_output="Machine learning is a subset of AI that learns from data.",
    retrieval_context=["ML is a branch of AI that uses algorithms to learn patterns."]
)

# Define metrics
relevancy_metric = AnswerRelevancyMetric(threshold=0.7)
faithfulness_metric = FaithfulnessMetric(threshold=0.7)

# Evaluate
results = evaluate(
    test_cases=[test_case],
    metrics=[relevancy_metric, faithfulness_metric]
)

print(results)
```

### Streaming Response Handler

```python
from openai import OpenAI

client = OpenAI()

def stream_response(prompt):
    stream = client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}],
        stream=True
    )
    
    for chunk in stream:
        if chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end="", flush=True)
    print()

stream_response("Write a short poem about AI.")
```

## Frequently Asked Questions

### What are Large Language Models and how do they work?
Large Language Models (LLMs) are neural networks trained on massive text datasets to understand and generate human-like text. They use the transformer architecture with self-attention mechanisms to process and generate sequences of tokens.

### How long does it take to learn LLMs?
With 1-2 lessons per day, you can complete this 100-lesson course in 10-14 weeks. Building production skills typically takes 3-6 months of hands-on practice.

### What prerequisites do I need for this course?
Basic Python programming and understanding of machine learning concepts are helpful. We recommend completing our [Python course](https://chatwhole.com/learn/python/) and [Machine Learning course](https://chatwhole.com/learn/machine-learning/) first.

### What is the difference between GPT-4 and open source LLMs like LLaMA?
GPT-4 is a proprietary model available only via API, while LLaMA is open source and can be run locally. GPT-4 generally performs better on complex tasks, but LLaMA offers more control, customization, and cost savings for production use.

### How do I fine-tune an LLM for my specific use case?
Start with [Fine Tuning LLMs](https://chatwhole.com/learn/llm/05-fine-tuning-llms/), then learn [LoRA and PEFT](https://chatwhole.com/learn/llm/06-lora-and-peft/) for efficient fine-tuning. Prepare your training data, choose a base model, and use Hugging Face Transformers or Unsloth for training.

### What is RAG and when should I use it?
RAG (Retrieval Augmented Generation) retrieves relevant documents at query time to provide context to the LLM. Use RAG when you need the model to reference specific, frequently updated information without fine-tuning.

### How do I reduce LLM inference costs?
Use quantization ([QLoRA](https://chatwhole.com/learn/llm/07-qlora-and-quantization/)), caching ([Prompt Cache](https://chatwhole.com/learn/llm/57-caching-strategies/)), batch inference, and smaller open-source models. See [Cost Optimization](https://chatwhole.com/learn/llm/58-cost-optimization/) for detailed strategies.

### What hardware do I need to run LLMs locally?
For 7B parameter models: 16GB RAM GPU (RTX 4090). For 13B: 24GB+ GPU. For 70B: Multiple GPUs or cloud instances. Quantized models (4-bit) reduce requirements significantly.

### How do I evaluate LLM performance?
Use [LLM Evaluation Benchmarks](https://chatwhole.com/learn/llm/15-llm-evaluation-benchmarks/) for standard metrics, [LLM Evaluation Frameworks](https://chatwhole.com/learn/llm/97-llm-evaluation-frameworks/) for custom evaluation, and human evaluation for quality assessment.

### What is prompt engineering and why is it important?
Prompt engineering is the practice of designing effective inputs to guide LLM behavior. Learn [Prompt Engineering](https://chatwhole.com/learn/llm/08-prompt-engineering/) and [Chain of Thought Reasoning](https://chatwhole.com/learn/llm/10-chain-of-thought-reasoning/) to get better results without fine-tuning.

### Can I use LLMs for commercial applications?
Yes! Many open-source LLMs have permissive licenses (Apache 2.0, MIT). Always check the specific model's license. See [Open Source vs Commercial](https://chatwhole.com/learn/llm/98-open-source-vs-commercial/) for guidance.
