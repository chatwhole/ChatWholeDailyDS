## Key Concepts Summary

### LLM Architecture

| Component | Description |
|-----------|-------------|
| Transformer | Self-attention based architecture |
| Tokenizer | BPE, SentencePiece, WordPiece |
| Embedding | Token + positional embeddings |
| Attention | Multi-head self-attention |
| Feed-forward | Two linear layers with activation |
| Layer Norm | Stabilizes training |

### Fine-Tuning Methods

| Method | Parameters Trained | VRAM Required | Quality |
|--------|-------------------|---------------|---------|
| Full Fine-Tuning | All | High | Best |
| LoRA | Low-rank matrices | Medium | Good |
| QLoRA | Quantized + LoRA | Low | Good |
| Prompt Tuning | Soft prompts | Very Low | Decent |
| Adapter | Small adapter layers | Low | Good |

### RAG Components

```
User Query → Retriever → Documents → LLM → Answer

Retriever: Vector DB (Pinecone, Chroma, Weaviate)
Documents: Chunks with embeddings
LLM: GPT-4, Claude, Llama
```

### Prompt Engineering Techniques

| Technique | Description |
|-----------|-------------|
| Zero-shot | No examples |
| Few-shot | Include examples |
| Chain-of-thought | Step-by-step reasoning |
| Self-consistency | Multiple samples + vote |
| Tree of thought | Explore multiple paths |

### Evaluation Metrics

| Metric | Use Case |
|--------|----------|
| Perplexity | Language modeling |
| BLEU | Translation |
| ROUGE | Summarization |
| F1 | QA tasks |
| Human Eval | Code generation |
| MMLU | General knowledge |
