## Key Concepts Summary

### Generative AI Model Types

| Type | Examples | Use Case | Strengths |
|------|----------|----------|-----------|
| LLM | GPT-4, Claude, LLaMA | Text generation | Versatile, reasoning |
| Diffusion | Stable Diffusion, DALL-E | Image generation | High quality images |
| GAN | StyleGAN, ProGAN | Image synthesis | Fast inference |
| VAE | VQ-VAE, beta-VAE | Anomaly detection | Smooth latent space |
| Multimodal | GPT-4V, Gemini | Text + Image | Cross-modal understanding |
| Audio | Whisper, Bark | Speech/Text | Multilingual support |

### RAG vs Fine-Tuning Decision Matrix

| Factor | RAG | Fine-Tuning |
|--------|-----|-------------|
| Setup time | Hours | Days-Weeks |
| Cost | Low | High |
| Data freshness | Real-time | Static |
| Accuracy | Good | Better |
| Hallucination | Lower | Higher |
| Best for | Dynamic data | Domain expertise |

### Fine-Tuning Approaches

| Method | Description | Compute Required | When to Use |
|--------|-------------|-----------------|-------------|
| Full FT | Update all parameters | Very High | Large dataset, high compute |
| LoRA | Low-rank adaptation | Medium | Medium dataset |
| QLoRA | Quantized LoRA | Low | Consumer GPUs |
| Prompt Tuning | Learnable prompts | Minimal | Quick experiments |
| ICL | In-context learning | None | No training needed |

### Agent Components

```
LLM + Tools + Memory + Planning = Agent

Tools: API calls, code execution, web search
Memory: Short-term (context window), Long-term (vector DB)
Planning: ReAct, Chain-of-thought, Tree-of-thought
```

### Key Generation Parameters

| Parameter | Range | Effect |
|-----------|-------|--------|
| Temperature | 0.0-2.0 | Controls randomness |
| Top-k | 1-100 | Limits sampling pool |

## Code Examples

### Basic Text Generation with Transformers

```python
from transformers import pipeline

# Initialize text generation pipeline
generator = pipeline("text-generation", model="gpt2")

# Generate text from a prompt
prompt = "The future of artificial intelligence is"
output = generator(prompt, max_length=50, num_return_sequences=1)
print(output[0]["generated_text"])
```

### RAG Implementation with LangChain

```python
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings
from langchain.llms import OpenAI
from langchain.chains import RetrievalQA

# Create vector store from documents
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(documents, embeddings)

# Create RAG chain
qa_chain = RetrievalQA.from_chain_type(
    llm=OpenAI(),
    retriever=vectorstore.as_retriever(search_kwargs={"k": 3})
)

# Query
result = qa_chain.run("What is the main topic of this document?")
```

### LoRA Fine-Tuning with PEFT

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b")

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
print(f"Trainable params: {model.print_trainable_parameters()}")
```

### Stable Diffusion Image Generation

```python
from diffusers import StableDiffusionPipeline
import torch

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    torch_dtype=torch.float16
).to("cuda")

image = pipe(
    "A sunset over mountains, digital art, highly detailed",
    num_inference_steps=50,
    guidance_scale=7.5
).images[0]

image.save("generated_image.png")
```

### Simple Agent with Tool Use

```python
from langchain.agents import initialize_agent, Tool
from langchain.llms import OpenAI

def search_web(query):
    return f"Results for: {query}"

tools = [
    Tool(name="Search", func=search_web, description="Search the web")
]

llm = OpenAI(temperature=0)
agent = initialize_agent(tools, llm, agent="zero-shot-react-description")
result = agent.run("What is the weather in New York?")
```

### Vector Database Setup with Chroma

```python
import chromadb
from chromadb.utils import embedding_functions

client = chromadb.Client()
collection = client.create_collection(
    name="documents",
    embedding_function=embedding_functions.OpenAIEmbeddingFunction()
)

# Add documents
collection.add(
    documents=["AI is transforming industries", "Machine learning is a subset of AI"],
    metadatas=[{"source": "article1"}, {"source": "article2"}],
    ids=["doc1", "doc2"]
)

# Query
results = collection.query(query_texts=["artificial intelligence"], n_results=2)
```

### Prompt Engineering Template

```python
def create_prompt(task, context, examples, input_text):
    return f"""Task: {task}

Context: {context}

Examples:
{chr(10).join(f'Input: {ex["input"]}\nOutput: {ex["output"]}' for ex in examples)}

Input: {input_text}
Output:"""

prompt = create_prompt(
    task="Classify sentiment",
    context="Product reviews",
    examples=[
        {"input": "Great product!", "output": "Positive"},
        {"input": "Terrible quality", "output": "Negative"}
    ],
    input_text="I love this item"
)
```

## Frequently Asked Questions

### What is Generative AI and how does it work?
Generative AI is a type of artificial intelligence that creates new content—text, images, audio, video, or code—by learning patterns from training data. It uses architectures like transformers, diffusion models, and GANs to generate novel outputs that resemble the training data.

### How long does it take to learn Generative AI?
With consistent study of 1-2 lessons per day, you can complete this 100-lesson course in 10-14 weeks. Building production skills typically takes 3-6 months of hands-on practice.

### What prerequisites do I need for this course?
Basic Python programming and understanding of machine learning concepts are helpful. We recommend completing our [Python course](https://chatwhole.com/learn/python/) and [Machine Learning course](https://chatwhole.com/learn/machine-learning/) first.

### What is the difference between RAG and fine-tuning?
RAG (Retrieval Augmented Generation) retrieves relevant documents at query time to provide context, while fine-tuning modifies the model's weights on custom data. RAG is better for dynamic, frequently updated data; fine-tuning is better for domain-specific expertise.

### How do I build a RAG application?
Start with [RAG System Design](https://chatwhole.com/learn/generative-ai/08-retrieval-augmented-generation/), then learn [Vector Databases](https://chatwhole.com/learn/generative-ai/09-vector-databases/) and [Advanced RAG Techniques](https://chatwhole.com/learn/generative-ai/68-advanced-rag-techniques/). Use LangChain or LlamaIndex for rapid prototyping.

### What are AI agents and how do they work?
AI agents are systems that use LLMs to reason, plan, and take actions using external tools. Learn about [Agent Architectures](https://chatwhole.com/learn/generative-ai/20-agent-architectures/), [Tool Use](https://chatwhole.com/learn/generative-ai/21-tool-use-function-calling/), and [Multi Agent Systems](https://chatwhole.com/learn/generative-ai/22-multi-agent-systems/).

### How do I reduce hallucinations in AI models?

### What is LoRA and why is it important for fine-tuning?
LoRA (Low-Rank Adaptation) is a parameter-efficient fine-tuning technique that adds small trainable matrices to frozen model weights. It reduces memory requirements by 60-80% while maintaining performance, making fine-tuning accessible on consumer GPUs.
