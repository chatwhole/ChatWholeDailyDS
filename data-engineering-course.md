## Key Concepts Summary

### Data Engineering Lifecycle

| Stage | Description | Key Technologies |
|-------|-------------|------------------|
| Generation | Data originates from source systems | Databases, APIs, logs, IoT sensors |
| Ingestion | Move data from sources to storage | Kafka, Flink, Airbyte, Debezium |
| Storage | Where data lives at rest | S3, GCS, Snowflake, BigQuery |
| Processing | Transform raw data into usable formats | Spark, dbt, Pandas, Flink |
| Serving | Make data available to consumers | Dashboards, APIs, ML feature stores |

### Data Warehouse Schema Comparison

| Schema | Description | Pros | Cons |
|--------|-------------|------|------|
| Star Schema | Denormalized dimensions around fact table | Fast reads, simple queries | Redundant storage |
| Snowflake Schema | Normalized dimension hierarchy | Storage efficiency | Complex joins |
| 3NF (Inmon) | Fully normalized enterprise model | Single source of truth | Long implementation |

### Pipeline Orchestration Tools Comparison

| Tool | Language | Scheduling | UI | Best For |
|------|----------|-----------|-----|----------|
| Apache Airflow | Python | Cron-based | Rich web UI | Complex DAGs, large teams |
| Prefect | Python | Dynamic | Modern UI | Python-native workflows |
| Dagster | Python | Asset-based | Asset lineage | Data-aware orchestration |
| Luigi | Python | Internal | Minimal | Simple batch jobs |

### Data Platform Comparison

| Platform | Type | Pricing Model | Best For |
|----------|------|---------------|----------|
| Snowflake | Cloud DW | Credits (compute) + storage | Elastic analytics, multi-cloud |
| BigQuery | Serverless DW | On-demand per TB scanned | Ad-hoc analytics, ML integration |
| Redshift | MPP DW | Node-based | AWS-native, large-scale |
| Databricks | Lakehouse | DBUs (compute) | Spark workloads, ML |
| Delta Lake | Open format | Storage only | ACID transactions on lakes |

### ETL vs ELT Comparison

| Aspect | ETL | ELT |
|--------|-----|-----|
| Transformation | Before loading | After loading |
| Target | Data warehouse | Data lake/lakehouse |
| Tools | Informatica, Talend | dbt, Spark, SQL |
| Best For | Small data, strict schemas | Large data, schema flexibility |

## Code Examples

### SQL: Window Functions for Data Engineers

```sql
-- Running total and day-over-day change
WITH daily_sales AS (
    SELECT
        order_date,
        SUM(amount) AS daily_revenue
    FROM orders
    GROUP BY order_date
)
SELECT
    order_date,
    daily_revenue,
    SUM(daily_revenue) OVER (
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total,
    daily_revenue - LAG(daily_revenue, 1) OVER (
        ORDER BY order_date
    ) AS day_over_day_change,
    ROUND(
        (daily_revenue - LAG(daily_revenue, 1) OVER (ORDER BY order_date))
        / NULLIF(LAG(daily_revenue, 1) OVER (ORDER BY order_date), 0) * 100,
        2
    ) AS pct_change
FROM daily_sales
ORDER BY order_date;
```

### Python: Complete ETL Pipeline

```python
import pandas as pd
from sqlalchemy import create_engine
from datetime import datetime
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class ETLPipeline:
    def __init__(self, source_conn, target_conn):
        self.source_engine = create_engine(source_conn)
        self.target_engine = create_engine(target_conn)

    def extract(self, query: str) -> pd.DataFrame:
        logger.info("Extracting data from source...")
        df = pd.read_sql(query, self.source_engine)
        logger.info(f"Extracted {len(df)} rows")
        return df

    def transform(self, df: pd.DataFrame) -> pd.DataFrame:
        logger.info("Transforming data...")
        df = df.drop_duplicates(subset=['order_id'])
        df = df[df['amount'] > 0]
        df['processed_at'] = datetime.utcnow()
        df['total'] = df['quantity'] * df['unit_price']
        logger.info(f"Transformed to {len(df)} rows")
        return df

    def load(self, df: pd.DataFrame, table_name: str):
        logger.info(f"Loading to {table_name}...")
        df.to_sql(table_name, self.target_engine,
                  if_exists='append', index=False, chunksize=1000)
        logger.info(f"Loaded {len(df)} rows")

    def run(self, extract_query: str, target_table: str):
        df = self.extract(extract_query)
        df = self.transform(df)
        self.load(df, target_table)

# Usage
pipeline = ETLPipeline(
    source_conn="postgresql://user:pass@source-db/production",
    target_conn="postgresql://user:pass@warehouse/analytics"
)
pipeline.run(
    "SELECT * FROM orders WHERE date >= '2024-01-01'",
    "orders_fact"
)
```

### Apache Airflow: Production DAG

```python
from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.operators.bash import BashOperator
from airflow.utils.task_group import TaskGroup

default_args = {
    "owner": "data-engineering",
    "retries": 3,
    "retry_delay": timedelta(minutes=5),
    "email_on_failure": True,
    "email": ["alerts@company.com"],
}

def extract(**context):
    import pandas as pd
    from sqlalchemy import create_engine
    engine = create_engine("postgresql://user:pass@source/db")
    df = pd.read_sql("SELECT * FROM orders", engine)
    context["ti"].xcom_push(key="row_count", value=len(df))
    return df.to_dict("records")

def transform(**context):
    import pandas as pd
    records = context["ti"].xcom_pull(task_ids="extract")
    df = pd.DataFrame(records)
    df = df.drop_duplicates(subset=["order_id"])
    df["processed_at"] = datetime.utcnow()
    return df.to_dict("records")

def load(**context):
    import pandas as pd
    from sqlalchemy import create_engine
    records = context["ti"].xcom_pull(task_ids="transform")
    df = pd.DataFrame(records)
    engine = create_engine("postgresql://user:pass@warehouse/db")
    df.to_sql("orders_fact", engine, if_exists="append", index=False)

with DAG(
    dag_id="daily_order_etl",
    default_args=default_args,
    schedule_interval="@daily",
    start_date=datetime(2024, 1, 1),
    catchup=False,
) as dag:
    extract = PythonOperator(task_id="extract", python_callable=extract)
    transform = PythonOperator(task_id="transform", python_callable=transform)
    load = PythonOperator(task_id="load", python_callable=load)
    extract >> transform >> load
```

### Apache Kafka: Producer and Consumer

```python
from kafka import KafkaProducer, KafkaConsumer
import json

# Producer
producer = KafkaProducer(
    bootstrap_servers=["kafka:9092"],
    value_serializer=lambda v: json.dumps(v).encode("utf-8"),
    acks="all",
    enable_idempotence=True
)

producer.send("orders.events", value={"order_id": "123", "amount": 99.99})
producer.flush()

# Consumer
consumer = KafkaConsumer(
    "orders.events",
    bootstrap_servers=["kafka:9092"],
    group_id="order-processing",
    auto_offset_reset="earliest",
    value_deserializer=lambda m: json.loads(m.decode("utf-8"))
)

for message in consumer:
    event = message.value
    print(f"Processing order: {event['order_id']}")
```

### Apache Spark: Optimized DataFrame ETL

```python
from pyspark.sql import SparkSession
from pyspark.sql import functions as F

spark = SparkSession.builder \
    .appName("ETL-Pipeline") \
    .config("spark.sql.adaptive.enabled", "true") \
    .config("spark.sql.autoBroadcastJoinThreshold", str(10 * 1024 * 1024)) \
    .getOrCreate()

# Read with schema enforcement
df = spark.read.parquet("s3://raw/orders")

# Filter early, broadcast join with dimension table
filtered = df.filter(F.col("amount") > 0)
customer_df = spark.read.parquet("s3://dim/customers")
enriched = filtered.join(F.broadcast(customer_df), "customer_id", "left")

# Write with partitioning
enriched.write.mode("overwrite") \
    .partitionBy("order_date") \
    .parquet("s3://curated/orders")
```
